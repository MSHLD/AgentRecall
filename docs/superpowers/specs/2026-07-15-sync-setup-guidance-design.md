# 同步初始化引导与开发版本更新状态设计

## 背景

当前开发模式仍把 `package.json` 版本与 GitHub 最新 Release 比较，因此通过 `npm run dev` 启动时会把源码开发构建显示成可更新的旧正式版。会话与 Skills 同步在 Supabase 表、字段、Storage bucket 或权限缺失时虽然能够复制 SQL，但没有直接进入 SQL Editor 的入口，首次配置也缺少完整步骤。两个同步弹窗的工具栏还会分成两行，远程会话弹窗高度不足，压缩了主要内容区域。

## 目标

- 开发构建不参与正式版自动更新，并清楚显示为开发版本。
- 首次配置只需运行一次组合 SQL，即可初始化会话和 Skills 同步。
- 单项故障仍提供对应功能的最新修复 SQL，避免要求用户重复初始化无关功能。
- 会话和 Skills 使用一致的数据库修复引导：复制 SQL、打开 SQL Editor、执行后刷新。
- 搜索、筛选、选择、上传、删除和刷新集中在顶部操作区。
- 远程会话弹窗充分利用窗口高度，列表成为主要可滚动区域。

## 开发版本更新行为

- Electron 未打包运行时返回明确的开发构建状态，不请求 GitHub Release。
- About 页显示“开发版本”，隐藏版本升级卡、立即更新、检查更新和自动检查开关。
- 已安装的打包应用保持现有自动检查、更新安装和失败兜底逻辑。
- CLI 的正式安装版本检查不受 Electron 开发模式影响。

## 首次配置

Remote sync 设置页增加统一的“初始化远程同步”引导：

1. 填写 Supabase Project URL 和 anon key。
2. 复制包含会话与 Skills 所需表、字段、索引、Storage bucket、显式权限和 RLS policy 的最新组合 SQL。
3. 打开当前项目的 Supabase SQL Editor；无法从 URL 识别项目时打开 Supabase Dashboard。
4. 粘贴并执行 SQL。
5. 返回应用刷新状态，再启用或使用同步功能。

组合 SQL 由现有两份幂等初始化 SQL 合并生成，允许首次初始化和后续升级重复执行。

## 故障修复引导

- 会话同步异常时复制会话 SQL；Skills 同步异常时复制 Skills SQL。
- 缺表、缺字段、缺 bucket 和权限不足统一呈现为面向用户的“Supabase 尚未准备完成”，原始技术错误作为次要详情展示。
- 引导卡固定提供“复制最新 SQL”“打开 SQL Editor”“刷新”三个动作。
- 数据库未恢复前禁用上传、下载和删除，避免制造新的失败请求。
- 未填写 URL/key 时引导用户先到设置配置；SQL 仍可复制，便于用户提前初始化项目。

## 工具栏与弹窗布局

### 会话同步

- 顶部操作区依次放置搜索、来源筛选、选择当前结果、上传到云端、删除云端副本和刷新。
- 桌面宽度下保持一行；窄窗口按完整控件分组换行，不压扁搜索框或按钮文字。
- Supabase 修复引导置于工具栏下方，恢复正常后消失。
- 弹窗默认使用约 88% 的可用窗口高度，并设置合理的最小高度；会话列表占据剩余空间并独立滚动。

### Skills

- Local 和 Remote 页均将刷新放在顶部操作区末尾，不再单独落到第二行。
- Local 页顶部依次放置搜索、来源筛选、排序、选择当前结果、上传选中和刷新。
- Remote 页顶部放置搜索、全选、下载全部、删除云端选中和刷新。
- Supabase 修复引导沿用会话同步的视觉与操作顺序。

## SQL Editor 链接

- 对标准 `https://<project-ref>.supabase.co` URL，打开 `https://supabase.com/dashboard/project/<project-ref>/sql/new`。
- URL 不完整或无法识别项目时，打开 `https://supabase.com/dashboard/projects`。
- 外部链接由主进程校验后使用系统浏览器打开，渲染进程不直接导航。

## 验收

- `npm run dev` 的 About 页只显示开发版本，不出现正式版更新提示。
- 打包应用仍能正常检查和安装 Release。
- 首次配置可从设置页复制组合 SQL并直接进入对应项目 SQL Editor。
- 会话缺少 `revision_version`、Skills 缺表以及权限不足时均出现相同结构的修复引导。
- 修复后点击刷新即可恢复同步界面，不需要关闭弹窗或重启应用。
- 常见桌面窗口中两个工具栏为一行，刷新按钮位于末尾。
- 会话弹窗明显高于当前实现，列表填满剩余空间；小屏下仍不超出视口。
